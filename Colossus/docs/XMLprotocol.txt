Colossus XML network / savegame event format

Conventions
-----------

Everything is case sensitive, unless stated otherwise.

[name]:         A string of up to 30 characters.
[player]:       The name of a player currently in the game
[colorname]:    One of "Black", "Brown", ...
[legion]:       A marker id like "Bk01".
[unit]:         A complete creature name like "Behemoth".
[units]:        One or more of [unit].
[land]:         A number of a masterboard hex, like "127".
[number]:       A number like "3".
[battlehex]:    A battle hex identifier like "C1"
[entryside]:    One of "right", "left", "bottom"
[bool]:         One of "true" or "false"


Requests (client to server)
--------

<join name="[name]">
    Join the game and pick a name.  Note that if multiple players pick the
    same name, the server may change it.  If no name is chosen, the server
    will pick one.

<observe name="[name]">
    Join as an observer.

<color color=[colorname]>
    Choose a color.

<start>
    Starts the game. No more players can connect.

<split parent=[legion] child=[legion]>
    <units>
        <creature1>
        <creaturen>
    </units>
</split>
    Split off [units] from [legion1] to a new legion with marker
    [legion2].

<merge parent=[legion] child=[legion]>
    Move all units from child legion back into parent legion.
    Used to cancel a split before movement is rolled, or just before
    DONEWITHMOVES if neither legion can move.

<roll />
    Finish splitting and wait for the movement roll.

<mulligan />
    Re-roll movement.

<move legion="[legion]" hex="[land]" {entryside="[entryside]" teleport="[bool"]} />
    Move [legion] to masterboard hex [land] entering on side
    [entryside].  If [entryside] is omitted, the server picks.
    If the legion can both teleport and move without teleporting and 
    teleport is omitted, the server picks.

<undomove legion="[legion]" />
    Move [legion] back to its previous location.

<recruit legion="[legion]" recruit="[unit]" {recruiter="[unit]"}
    Muster a recruit using the right number of recruiter.  If recruiter is not
    set then the server picks.

<undorecruit legion="[legion]" />
    [legion] undoes its last recruit. 

<engage hex="[land]" />
    Resolve the engagement in [land].

<flee legion="[legion]" />
    [legion] flees.

<donotflee legion="[legion]" />
    [legion] does not flee.

<concede legion="[legion]" />
    [legion] concedes the battle.

<donotconcede legion="[legion]" />
    [legion] does not concede the battle.

<negotiate hex="[land]" survivor="[legion]">
    <units>
        <creature1>
        <creatureN>
    </units>
</negotiate>
    Offer a negotiated settlement to the engagement in [land] where 
    survivor wins, and is left with units.  If survivor or units is
    missing or empty, offer mutual destruction.

<accept hex="[land]" survivor="[legion]">
    <losses legion="[legion]">
        <creature1>
        <creatureN>
    </losses>
    <losses legion="[legion]">
        <creature1>
        <creatureN>
    </losses>
</accept>
    Accept a settlement proposed by the other player.

<fight hex="[land]" />
    Cease negotiations in [land] and fight a battle.

<enter creature="[unit]" hex="[battlehex]" />
    Move creature from offboard to hex.

<maneuver starthex="[battlehex]" endhex="[battlehex2]" />
    Move unit in starthex to endhex.

<undomaneuver hex="[battlehex]" />
    Move unit in hex back to its previous location.

<donemaneuvering />
    All battle moves are finished.  Move on to strike phase, or the
    opponent's battle turn, as appropriate.

<strike strikerhex="[battlehex]" targethex="[battlehex]" 
        dice="[number]" strike="[number]" />
    Unit in strikerhex strikes unit in targethex, rolling dice against the 
    given strike number.  The server will assign dice and strike if missing.

<carry targethex="[battlehex]" hits="[number]" />
    Carry hits to the unit in targethex.

<donestriking />
    All battle strikes are finished.  Move on to the opponent's phase.

<summon summoner="[legion1]" donor="[legion2]" creature="[unit]" />
    Summoner summons a unit from donor.

<acquire legion="[legion]" creature="[unit]" />
    Legion acquires a unit.

<nextturn />
    Finish this player's entire game turn.

<withdraw />
    This player withdraws from the entire game.

<replay />
    Resend the stream of all significant events so far in this game to
    this client.



Messages (server to one or more clients)
--------

# TODO All options
<formgame variant="[variantname]" minplayers="[number]" maxplayers="[number]" />

<nak packet="[number]" />
    Request [number] (starting from 0) from this client was denied.

<join name="[name]" />
    A player with name [name] has connected.

<color name="[name]" color=[colorname]>
    Player has chosen color. 

<tower name="[name]" hex="[land]" />
    Player has been assigned tower.

<split parent="[legion]" child="[legion]" numsplit="[number]">
    <units>
        <creature1>
        <creaturen>
    </units>
</split>
    A player splits off [number] units from [legion1] to a new
    legion with marker [legion2].  (The splitoff units will only 
    be identified if this client is allowed to see the contents
    of this stack.)

<merge parent=[legion] child=[legion]>
    child was merged back into parent.

<roll die="[number]" />
    The active player rolled for movement. 

<mulligan />
    The active player took a mulligan.

<move legion="[legion]" hex="[land]" entryside="[entryside]" teleport="[bool"] />
    [legion] moved to masterboard hex [land] entering on side
    [entryside].  If teleport is "true" then this move was a teleport.

<undomove legion="[legion]" />
    [legion] moved back to its previous location.

<engage hex="[land]" />
    <units>
        <creature1>
        <creaturen>
    </units>
    Active player chose to resolve the engagement in [land].  The opposing legion's
    units are revealed.

<flee legion="[legion]">
    <units>
        <creature1>
        <creaturen>
    </units>
</flee>
    [legion] fled, losing units.

<donotflee legion="[legion]" />
    [legion] did not flee.

<concede legion="[legion]">
    <units>
        <creature1>
        <creaturen>
    </units>
</concede>
    [legion] conceded the battle, losing units.

<donotconcede legion="[legion]" />
    [legion] did not concede the battle.

<negotiate hex="[land]" survivor="[legion]">
    <units>
        <creature1>
        <creatureN>
    </units>
</negotiate>
    An offer was made, with survivor (if any) winning and
    keeping units.

<accept hex="[land]" survivor="[legion]">
    <losses legion="[legion]">
        <creature1>
        <creatureN>
    </losses>
    <losses legion="[legion]">
        <creature1>
        <creatureN>
    </losses>
</accept>
    An offer was accepted.  Each legion's losses are shown.

<fight hex="[land]" />
    A battle will be fought.

<enter creature="[unit]" hex="[battlehex]" />
    Unit moved from offboard to hex.

<maneuver starthex="[battlehex]" endhex="[battlehex2]" />
    Unit moved from starthex to endhex.

<undomaneuver hex="[battlehex]" />
    Unit in hex moved back to its previous location.

<donemaneuvering />
    All battle moves are finished.  Move on to strike phase, or the
    opponent's battle turn, as appropriate.

<drift  [battlehex] />
    Unit in [battlehex] took one point of drift damage.

<strike strikerhex="[battlehex]" targethex="[battlehex]" 
        dice="[number]" strike="[number]" hits="[number]" carries="[number]">
    <rolls>
        [number1]
        [numberN]
    </rolls>
    <carryhexes>
        [battlehex1]
        [battlehexN]
    </carryhexes>
</strike>
    Unit in strikerhex struck unit in targethex, rolling dice against the 
    given strike number.  Exact rolls, hits, carries, and carry targets 
    are given.

<carry targethex="[battlehex]" hits="[number]" carries="[number]">
    <carryhexes>
        [battlehex1]
        [battlehexN]
    </carryhexes>
</carry>
    Hits were carried to the unit in targethex.  If there are still carries
    and carry targets remaining, they are shown.

<donestriking />
    All battle strikes are finished for this phase.

<summon summoner="[legion1]" donor="[legion2]" creature="[unit]" />
    Summoner summoned a unit from donor.

<acquire legion="[legion]" creature="[unit]" />
    Legion acquired a unit.

<points player="[player]" points="[number]" />
    [player] receives [number] points

<eliminateplayer loser="[player]" slayer="[player]" halfpoints="[number]" />
    loser has been eliminated from the game by slayer (may be null).

<gameover winner="[player]" />
    The game is over.  If no winner is given, the game is a draw.

<recruit legion="[legion]" recruit="[unit]" recruiter="[unit]" />
    Muster a recruit using the right number of recruiter.

<undorecruit legion="[legion]" />
    [legion] undoes its last recruit. 

<turn player="[player]" number="[number]" />
    Start of a new turn.
